这个仓库里有一个小型 C++ tiny C frontend。你的任务是为它的命令行行为添加黑盒 pytest 测试套件。

不要修复或重构实现。只添加测试。

## 需要测试的 CLI 规格

项目通过以下命令构建：

```bash
cmake -S . -B build
cmake --build build
```

构建结果必须包含名为 `cparser` 的可执行文件。

可执行文件从 stdin 读取完整输入，并支持：

- `cparser --tokens`
  - 每行输出一个 token，格式为 `TYPE:lexeme`
  - token 类型包括 `KEYWORD`、`IDENT`、`INT`、`OP`、`LPAREN`、`RPAREN`、`LBRACE`、`RBRACE`、`SEMI`、`COMMA`
  - 支持的运算符包括 `+`、`-`、`*`、`/`、`=`、`==`、`<`、`<=`、`>`、`>=`、`!=`
- `cparser --parse`
  - 将 stdin 当作整数算术表达式解析
  - 成功时只输出 `OK`，并以退出码 0 退出
  - 格式错误时以非 0 退出码退出，并向 stderr 输出 `ERROR:` 信息
- `cparser --eval`
  - 对包含括号、`+`、`-`、`*`、`/` 的整数算术表达式求值
  - 必须实现正常优先级和左结合
  - 成功时输出整数结果

空白字符包括空格、tab 和换行。`//` 行注释和 `/* ... */` 块注释应被忽略。格式错误的输入不能卡死。

## 要求

- 在 `tests/` 下添加 pytest 测试。
- 测试应在临时或本地 build 目录中构建项目，并通过 subprocess 调用 `cparser` 可执行文件。
- 保持黑盒测试：断言 CLI 行为，不要依赖私有实现细节。
- 不要修改 `main.cpp`、`CMakeLists.txt` 或其他实现文件。
- 你的测试暴露当前实现中的 bug 是可以的。
- `python3 test_basic.py` 只用于检查 pytest 能否发现你的测试。
